# Codex Build Prompt — 预警出险模块重建

你将在仓库 `patshin/risk-assistant-demo` 中重建“预警出险”模块。

本任务不是在旧模块上做表面美化。新高保真 UI、已确认系统性设计和 `design-references/warning-default/DESIGN_SPEC.md` 是新实现的事实来源。

旧预警出险页面结构、组件、数据类型、Mock、状态管理和CSS均不构成兼容约束。允许根据新设计彻底重建该模块。

## 九、业务规则

必须实现：

1. 客户户数按法人公司去重。
2. 法人客户可有所属集团名称。
3. 所属集团不参与法人客户数量计数。
4. 一个法人客户存在任一出险资产时，只计入出险客户数。
5. 客户名下资产金额按每笔资产自身状态归类。
6. 出险资产与当前预警状态互斥。
7. 一级、二级、重大预警可升级、降级、解除和重新进入。
8. 进入重大预警，包括从二级升级到重大。
9. 预警原因缺失时显示明确空值，不生成原因。
10. 跟踪状态不等于处置状态。
11. 加入跟踪不得改变风险状态。
12. 逾期9天按到期日与出险认定日计算，不使用当前日期。

不确定口径不得由前端自行推断。
