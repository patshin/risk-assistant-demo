# Codex Acceptance & Repair Prompt — 预警出险模块

你现在进行“预警出险”模块的第二轮严格验收与修复。

这不是只运行 `npm run build` 的检查。你必须在真实浏览器中逐页验证视觉、业务数据、状态语义、交互动线、响应式和控制台状态；发现问题后直接修复，再重新截图验证。

1. 读取仓库根目录及相关目录中的全部 `AGENTS.md`。

   - `design-references/warning-default/DESIGN_SPEC.md`
   - `design-references/warning-default/` 中全部PNG

   - `git status --short`
   - `git branch --show-current`

   - 不覆盖

5. 检查本次实现的文件范围。
6. 不假设当前实现正确。
7. 不新增依赖，除非先获得批准。

- 新增重大预警0.60亿元、5户

- AI预测客户出险
- 与当前数据不一致的数字

## 八、数据状态验收

逐一构造并验证：

### Loading

- 页面骨架尺寸接近最终内容
- 不发生大幅布局跳动
- 不使用0表示加载

### Empty

- 无风险客户
- 无状态记录
- 无历史趋势
- 无跟踪记录

应显示明确空状态和可行操作。

### No Results

- 搜索或筛选无结果
- 显示当前条件
- 提供重置筛选
- 不显示空白页面

### Error

- 页面请求失败
- 图表请求失败
- 跟踪提交失败
- 报告加入失败
- Copilot查询失败

必须：

- 显示错误
- 支持重试
- 保留表单数据
- 不导致白屏

### Zero

真实零显示：

- `0.00亿元`
- `0户`

### Unavailable

数据不可用显示：

- `—`
- `暂无数据`

不得混用。

### Extreme

- 所有预警均为重大
- 单一等级为0
- 百分比和不为100
- 趋势只有单点
- 趋势极端波动

不得溢出或崩溃。
